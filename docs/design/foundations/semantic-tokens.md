# Semantic Tokens

This document explains the functional meaning of the design tokens.

## Color Semantics

- **Background:** High-energy gradient (`primary-gradient`) for the main workspace.
- **Surface:** White/Light (`#fff`) for interactive areas (Cards, Modals).
- **Text:** High contrast for headings (`primary`), subtle for metadata (`secondary`).
- **Feedback:** Standard mapping for CRUD and validation states (Success/Error/Warning).

## Typography Hierarchy

- **Primary (Sans):** Optimized for readability and productivity.
- **Monospace (Mono):** Reserved for technical data, IDs, and code samples.
